
# 项目状态与规划

本页不发布版本日期，也不承诺未来 provider 或平台。此类内容比代码更快过期，并需要产品负责人决定。

## 当前已实现的范围

当前仓库包含：

- 主服务、记忆服务以及 agent/tool 服务；
- 浏览器与 Electron 桌面路由；
- Live2D、VRM、MMD、PNGTuber 和桌宠相关资源/功能；
- 基于角色的 provider 配置与多条语音路径；
- 持久化记忆投影、召回、人格、反思/证据以及事件/outbox 维护；
- 插件 SDK、内置插件、托管插件 UI 和打包门禁；
- Docker 与跨平台桌面构建 workflow。

代码中存在某项实现，并不保证每个 provider、平台或分发方式获得同等支持。

## 有效计划在哪里

按以下顺序查找：

1. 已接受的 GitHub issue 与关联设计记录；
2. 当前 milestone 或 project board（如果有维护）；
3. 已合并 PR 和 release note，用于确认已完成工作；
4. 代码和测试，用于确认实际当前行为。

实现 roadmap 项目前先向维护者确认范围。不要从旧目标日期、版本标签、营销文案或推测性的“阶段”描述中推导授权。

## 更新本页

只有存在权威公开规划来源和负责人时，才写入带日期或版本的承诺。优先链接来源，不要复制细节；已完成或取消的承诺应及时删除。
