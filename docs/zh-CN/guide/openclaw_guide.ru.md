
# Подключение N.E.K.O. к QwenPaw

Для совместимости с существующими настройками интеграция QwenPaw в N.E.K.O. по-прежнему называется **OpenClaw**. Переключатель OpenClaw подключается к отдельно запущенному сервису QwenPaw.

## 1. Проверьте источник и установите QwenPaw

Сверяйтесь с актуальными инструкциями в [официальном репозитории QwenPaw](https://github.com/agentscope-ai/QwenPaw). Команды ниже загружают и сразу выполняют удалённый установочный скрипт. Если того требует политика безопасности, сначала изучите скрипт. Установка может быть заблокирована в ограниченной сети или на управляемом устройстве.

macOS / Linux:

```bash
curl -fsSL https://qwenpaw.agentscope.io/install.sh | bash
```

Windows PowerShell:

```powershell
irm https://qwenpaw.agentscope.io/install.ps1 | iex
```

Установщик подготавливает `uv`, изолированное окружение, QwenPaw и зависимости. После установки откройте новый терминал.

## 2. Инициализация

```bash
qwenpaw init --defaults
```

Перед подтверждением прочитайте предупреждение QwenPaw. Локальный экземпляр получает доступ к файлам, командам и учётным данным, доступным учётной записи процесса. Не предоставляйте один экземпляр пользователям, которые не доверяют друг другу.

![Предупреждение безопасности при инициализации QwenPaw](assets/openclaw_guide/image1.png)

## 3. Запуск и проверка

```bash
qwenpaw app
```

Консоль по умолчанию доступна по адресу `http://127.0.0.1:8088/`. Не закрывайте терминал и откройте адрес в браузере. Если страница не загружается, сначала устраните ошибку запуска QwenPaw, а затем включайте интеграцию N.E.K.O.

Не публикуйте сервис за пределами localhost, пока не настроите аутентификацию и сетевую границу.

## 4. Настройка модели в QwenPaw

В консоли QwenPaw откройте страницу моделей, выберите provider, укажите необходимые учётные данные и сохраните настройки. Затем выберите настроенную модель на странице чата. Доступные providers и модели зависят от установленной версии QwenPaw, поэтому ориентируйтесь на текущий интерфейс, а не на скопированный список.

![Настройка модели QwenPaw](assets/openclaw_guide/image2.png)

## 5. Необязательно: persona исполнителя

В [архиве замены](assets/openclaw_guide/qwenpaw-executor-profile.zip) находятся `SOUL.md`, `AGENTS.md` и `PROFILE.md` для профиля исполнителя. Этот шаг не нужен для подключения и изменяет поведение QwenPaw.

Перед заменой:

1. остановите QwenPaw и сохраните резервную копию `.qwenpaw/workspaces/default`;
2. изучите архив и сравните его с текущим workspace;
3. копируйте только те файлы, которые действительно хотите заменить.

Каталог настроек обычно находится в `%USERPROFILE%\.qwenpaw` на Windows или `~/.qwenpaw` на macOS/Linux. Удаление `BOOTSTRAP.md` относится только к необязательной настройке этого профиля и не требуется для подключения к N.E.K.O. После изменений снова запустите `qwenpaw app`.

## 6. Включение в N.E.K.O.

1. Запустите QwenPaw и оставьте его работающим.
2. Откройте панель paw/Agent в N.E.K.O.
3. Включите главный переключатель Agent.
4. Включите дочерний переключатель **OpenClaw**.
5. Дождитесь проверки доступности.

По умолчанию N.E.K.O. подключается к `http://127.0.0.1:8088`. Если QwenPaw использует другой адрес, измените `openclawUrl` в основной конфигурации N.E.K.O. и повторите попытку.

Текущий adapter распознаёт console API QwenPaw v2 и прежние agent-compatible API. Проверка доступности обращается к `/api/version` или `/api/agent/health` в зависимости от варианта, после чего используется подходящий console или agent endpoint. Для стандартной console-конфигурации отдельный файл channel не нужен.
