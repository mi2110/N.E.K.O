# Project Status and Planning

This page does not publish version dates or promise future providers/platforms. Those claims become stale faster than the code and require product-owner decisions.

## Current implemented surfaces

The current repository contains:

- main, memory, and agent/tool services;
- browser and Electron desktop routes;
- Live2D, VRM, MMD, PNGTuber, and desktop-pet-related assets/features;
- role-based provider configuration and multiple voice paths;
- persistent memory projections, recall, persona, reflection/evidence, event/outbox maintenance;
- plugin SDK, built-in plugins, hosted plugin UI, and packaging gates;
- Docker and cross-platform desktop build workflows.

Implementation presence is not a guarantee that every provider, platform, or distribution is equally supported.

## Where active plans live

Use, in order:

1. accepted GitHub issues and linked design records;
2. the current milestone/project board, when maintained;
3. merged PRs and release notes for completed work;
4. the code and tests for actual current behavior.

Before implementing a roadmap item, confirm scope with maintainers. Do not infer authorization from old target dates, version labels, marketing copy, or speculative “phase” descriptions.

## Updating this page

Only add a dated/versioned commitment when there is an authoritative public planning source and an owner. Prefer linking that source over duplicating details here; remove completed or cancelled claims promptly.
